# Mobile Memo Click Fix - Touch Event Double-Firing

## Problem
در نسخه موبایل/تبلت، با تاچ کلیک اول همان لحظه لینک ممو باز می‌شد و سیستم دو کلیک کار نمی‌کرد.

## Root Cause
در دستگاه‌های تاچ، هر دو event زیر فعال می‌شدند:
1. `touchend` event
2. `click` event (synthetic click که مرورگر بعد از touch ایجاد می‌کند)

این باعث می‌شد که `handleClick` دو بار پشت سر هم اجرا شود:
- **کلیک اول**: zoom می‌شد و state روی `true` می‌رفت
- **کلیک دوم (300ms بعد)**: لینک باز می‌شد

## Solution Implemented

### 1. Click Debouncing ⏱️
اضافه کردن یک سیستم debounce با 300ms تاخیر:
```javascript
let lastClickTime = 0;
const CLICK_DEBOUNCE = 300; // 300ms debounce

const handleClick = (clientX, clientY, isTouchEvent = false) => {
  const now = Date.now();
  if (now - lastClickTime < CLICK_DEBOUNCE) {
    return; // Ignore rapid successive clicks
  }
  lastClickTime = now;
  // ... rest of click handling
}
```

### 2. Separate Event Listeners 📱
جدا کردن event listener ها برای موبایل و دسکتاپ:

**Desktop (Mouse):**
- `mousemove` → hover effects
- `click` → click handling

**Mobile (Touch):**
- `touchmove` → hover effects (tablets only)
- `touchend` → click handling
- ❌ `click` event disabled to prevent double-firing

### 3. Touch Event Flag 🏴
اضافه کردن پارامتر `isTouchEvent` برای تشخیص نوع event:
```javascript
handleClick(touch.clientX, touch.clientY, true); // Touch
handleClick(event.clientX, event.clientY, false); // Mouse
```

```javascript
  window.addEventListener('touchend', onTouchEnd, false);

  window.addEventListener('click', onMouseClick, false);

```

#### Change 4: Updated Cleanup
```javascript
cleanup: () => {
  if (isMobile) {
    window.removeEventListener('touchend', onTouchEnd);
  } else {
    window.removeEventListener('click', onMouseClick);
  }
}
```

## How It Works Now

### Desktop (Mouse) 🖱️
1. **First click**: Zoom to memo
2. **Second click**: Open link
3. Click events handled by `click` listener

### Mobile (Touch) 📱
1. **First tap**: Zoom to memo (via `touchend`)
2. **Second tap**: Open link (via `touchend`)
3. Synthetic `click` event is never registered
4. Debouncing prevents accidental double-taps

### Debounce Protection 🛡️
- Any clicks within 300ms of previous click are ignored
- Prevents:
  - Touch + synthetic click double-firing
  - Accidental double-taps
  - Rapid successive clicks

## Notes
- 300ms debounce is standard for preventing double-tap zoom on mobile
- Synthetic click events are automatically generated by browsers after touch
- This fix ensures clean separation between touch and mouse interactions
- WeakMap in hotspots.js handles the two-click state tracking
- Debouncing is a safety layer to prevent any edge cases
